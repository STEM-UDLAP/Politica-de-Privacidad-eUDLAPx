Última actualización: Diciembre 2025

Introducción

Este documento informa de manera específica para qué la aplicación móvil eUDLAPx recopila, utiliza, almacena y/o transmite datos personales de sus usuarios, incluyendo el uso de tecnologías biométricas (detección de vida / liveness) para autenticación y seguridad, cuando el usuario decide habilitar la funcionalidad de acceso al campus dentro de la aplicación.

eUDLAPx es una aplicación desarrollada para la Fundación Universidad de las Américas, Puebla (UDLAP). El tratamiento de datos personales se realiza en cumplimiento del Aviso de Privacidad Institucional de UDLAP y la legislación mexicana aplicable. Este documento complementa (no sustituye) dicho Aviso y describe finalidades y tratamientos específicos de eUDLAPx.

Aclaración sobre el uso de la app

Si usted no otorga el consentimiento expreso para el tratamiento de datos biométricos (detección de vida / liveness), no podrá utilizar la funcionalidad de acceso al campus dentro de la aplicación eUDLAPx, sin que ello limite otros mecanismos institucionales de acceso.

Finalidades del Tratamiento en eUDLAPx

En el contexto de eUDLAPx, sus datos personales podrán tratarse para las siguientes finalidades:

Finalidades primarias (necesarias para el servicio)

  • Autenticación de identidad del usuario dentro de la aplicación.
  • Habilitar y operar la funcionalidad de acceso al campus dentro de la aplicación cuando el usuario la activa.
  • Prevención de fraude y suplantación de identidad mediante detección de vida (liveness).
  • Seguridad, auditoría y trazabilidad: generación de bitácoras, registros y evidencias técnicas.
  • Soporte técnico, atención a incidentes y mejora de estabilidad/seguridad de la app.

Finalidades secundarias

eUDLAPx no utiliza los datos biométricos para finalidades secundarias (por ejemplo, mercadotecnia o publicidad). En caso de que eUDLAPx llegara a utilizar datos personales (no biométricos) para finalidades secundarias, se informará y recabará, cuando aplique, el consentimiento correspondiente, conforme al Aviso de Privacidad Institucional.

Compatibilidad con el Aviso de Privacidad Institucional

El Aviso de Privacidad Institucional de UDLAP contempla transferencias/remisiones como: (i) prestadores de servicios contratados para tratamiento de datos; (ii) otras instituciones educativas para proyectos académicos, culturales o deportivos; y (iii) uso de nombre/domicilio/teléfono/correo para envío de información sobre servicios y eventos.

En eUDLAPx, las finalidades descritas en este documento se enfocan en autenticación y seguridad. Si alguna actividad de eUDLAPx implicara una finalidad adicional no contemplada aquí o no alineada con el Aviso Institucional, este documento deberá actualizarse antes de su implementación.

Datos Personales Sensibles y Consentimiento Expreso

La detección de vida (liveness) implica el tratamiento de datos biométricos, los cuales, en términos de la LFPDPPP, se consideran datos personales sensibles. Por ello, su tratamiento requiere que el consentimiento sea expreso.

¿Cómo se obtiene el consentimiento expreso?

  1. Antes de activar “Acceso al campus” dentro de la app, eUDLAPx muestra este documento y un aviso específico de biométricos (liveness), indicando datos, finalidades y alcance.
  2. El usuario otorga su consentimiento mediante una acción afirmativa (por ejemplo: marcar una casilla “Acepto el tratamiento de datos biométricos para liveness” y pulsar “Continuar”).
  3. eUDLAPx registra la aceptación (fecha/hora, versión del aviso y usuario) como evidencia de consentimiento.
  4. Sin esta aceptación, eUDLAPx no habilita el liveness ni la funcionalidad de acceso al campus dentro de la aplicación.

Adicionalmente, en iOS/Android se solicitarán permisos del sistema (p. ej., cámara). Estos permisos no sustituyen el consentimiento expreso; son un requisito técnico adicional.

Tecnologías Biométricas en iOS: TrueDepth API

¿Qué es TrueDepth API?

En dispositivos iOS, eUDLAPx utiliza la tecnología TrueDepth API de Apple para realizar detección de vida (liveness). Esta tecnología verifica que una persona física real está presente durante el proceso de autenticación.

Datos Procesados con TrueDepth

Durante el proceso de liveness, la aplicación procesa:

  • Patrones de movimiento facial
  • Parpadeos
  • Mediciones de distancia
  • Orientación de cabeza (Yaw, Pitch, Roll)

Finalidad

Los datos biométricos se utilizan exclusivamente para:

  • Detectar vida (liveness)
  • Prevenir suplantación de identidad
  • Habilitar y proteger la funcionalidad de acceso al campus dentro de la aplicación

Consentimiento (expreso)

El uso de TrueDepth requiere consentimiento expreso previo a su utilización. Este consentimiento se obtiene dentro de la app mediante una acción afirmativa (por ejemplo, casilla de aceptación), antes de solicitar el permiso de cámara/Face ID del sistema. Si usted no lo otorga, no podrá utilizar la funcionalidad de acceso al campus dentro de eUDLAPx.

Proceso de Detección de Vida

Liveness Activo (en dispositivo):

  1. Procesamiento completamente local
  2. Verificación en tiempo real
  3. Resultado booleano
  4. Datos crudos descartados inmediatamente

Liveness Pasivo (en servidor UDLAP):

  1. Envío de vector matemático no reconstructible
  2. Análisis por modelo de ML
  3. No se envían imágenes
  4. Vector eliminado tras la verificación

Tecnologías Biométricas en Android: Google ML Kit

¿Qué es Google ML Kit?

En dispositivos Android, eUDLAPx utiliza ML Kit Vision de Google para detección de vida (liveness). Esta tecnología emplea algoritmos de visión computacional para detectar presencia humana y descartar intentos de fraude.

Datos Procesados con ML Kit

ML Kit procesa temporalmente, sin almacenar:

  • Movimientos faciales básicos
  • Parpadeos y presencia de rostro
  • Orientación general del rostro
  • Características matemáticas generadas por ML Kit

No se almacenan imágenes o videos.

Finalidad

  • Verificar presencia real (liveness)
  • Evitar suplantación con fotos o videos
  • Habilitar y proteger la funcionalidad de acceso al campus dentro de la aplicación

Consentimiento (expreso) y permisos

Para usar liveness en Android se requiere: (i) consentimiento expreso dentro de la app antes de su activación, y (ii) permiso de cámara del sistema operativo. Sin el consentimiento expreso y/o sin el permiso de cámara, no podrá utilizar la funcionalidad de acceso al campus dentro de eUDLAPx.

Proceso de Liveness con ML Kit

Liveness en dispositivo (Android):

  1. ML Kit analiza imágenes en memoria temporal
  2. No se capturan ni almacenan fotografías
  3. Se produce un resultado booleano
  4. Los datos se descartan en milisegundos

Liveness Pasivo (en servidor UDLAP):

  1. Generación de vector matemático no reconstructible
  2. Envío temporal al servidor
  3. Análisis por modelo de ML
  4. Eliminación tras la verificación

Procesamiento y Almacenamiento de Datos

Procesamiento en Dispositivo (iOS y Android)

Los datos biométricos utilizados para liveness (TrueDepth en iOS y ML Kit en Android) se procesan principalmente en su dispositivo. Ninguna imagen facial, mapa de profundidad ni video se almacena como parte del funcionamiento ordinario de liveness.

Información Transmitida a Servidores UDLAP

La aplicación transmite únicamente lo necesario para operar la autenticación y auditoría:

  • Resultado booleano de verificación de vida
  • Vector matemático no reconstructible (cuando aplique)
  • Marca de tiempo
  • ID de usuario

Información Almacenada

  • Resultado de autenticación
  • Marca de tiempo
  • ID de usuario
  • Registros de auditoría

Información que NUNCA se Almacena ni Comparte

  • Mapas de profundidad o modelos 3D
  • Datos crudos del sensor TrueDepth
  • Imágenes o fotografías faciales
  • Videos capturados por la cámara
  • Plantillas biométricas reconstruibles

Compartir con Terceros, Transferencias y Remisiones

Datos biométricos (liveness): eUDLAPx no transfiere ni remite datos biométricos a terceros. El tratamiento biométrico para detección de vida (liveness) se realiza exclusivamente en el dispositivo del usuario y/o en servidores institucionales de la UDLAP, bajo su control directo.

Ningún proveedor externo, encargado o tercero tiene acceso a datos biométricos, imágenes, videos, mapas de profundidad ni vectores biométricos derivados.

Para otros datos personales no biométricos (por ejemplo, infraestructura tecnológica, autenticación o soporte), la UDLAP podrá utilizar prestadores de servicios que actúan como encargados, conforme al Aviso de Privacidad Institucional, bajo obligaciones de confidencialidad y seguridad, sin que ello implique transferencia o remisión de datos biométricos.

Otros Datos Personales Recopilados

Además de los datos biométricos descritos (cuando se habilita liveness), eUDLAPx puede tratar:

  • Datos de autenticación: MSAL (Microsoft) para inicio de sesión
  • Registros de acceso al campus dentro de la app (resultado, fecha/hora, identificadores)
  • Datos de dispositivo: modelo, sistema operativo, token FCM
  • Datos de perfil: nombre, ID, rol

Retención de Datos

Datos biométricos para liveness

Retención: no se almacenan imágenes/biometría cruda; el procesamiento es temporal. Los vectores matemáticos (cuando aplique) se eliminan tras la verificación (segundos).

Otros Datos de Autenticación

Resultados booleanos y bitácoras: 2 años desde el último acceso (o el plazo que resulte aplicable por normativa interna y/o legal).

Medidas de Seguridad

  • Cifrado TLS/HTTPS
  • Keychain (iOS) y Encrypted Shared Preferences (Android)
  • Secure Enclave en dispositivos compatibles
  • Auditorías regulares y controles de acceso

Derechos ARCO

Acceso, rectificación, cancelación u oposición pueden solicitarse siguiendo el Aviso de Privacidad Institucional.

Revocación del Consentimiento

Usted puede revocar su consentimiento para el tratamiento de datos biométricos deshabilitando la funcionalidad de acceso al campus dentro de la app (cuando la app lo permita) y/o retirando permisos del sistema. La revocación puede implicar que no pueda usar la funcionalidad de acceso al campus dentro de eUDLAPx.

En iOS

  1. Configuración > eUDLAPx > Cámara
  2. Desactivar acceso a cámara (TrueDepth)

En Android

  1. Ajustes > Aplicaciones > eUDLAPx
  2. Permisos > Cámara > Denegar

Desinstalar la Aplicación

Elimina permisos y datos locales automáticamente.
