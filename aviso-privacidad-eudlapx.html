Introducción

Este documento describe cómo la aplicación móvil eUDLAPx recopila, procesa y protege los datos personales de sus usuarios, con especial énfasis en el uso de tecnologías biométricas para autenticación y seguridad.

eUDLAPx es una aplicación desarrollada para la Fundación Universidad de las Américas, Puebla (UDLAP). El tratamiento de datos personales se realiza en cumplimiento del Aviso de Privacidad Institucional de UDLAP y la legislación mexicana aplicable.

Uso de TrueDepth API y Detección de Vida (Liveness)

¿Qué es TrueDepth API?

eUDLAPx utiliza la tecnología TrueDepth API de Apple para realizar detección de vida (liveness) durante la autenticación de usuarios. Esta tecnología verifica que una persona física real está presente durante el proceso de autenticación.

Datos Recopilados con TrueDepth

Durante el proceso de autenticación, la aplicación recopila los siguientes datos mediante TrueDepth API:

  • Patrones de movimiento facial (giros de cabeza, asentimientos)
  • Detección de parpadeo de ojos
  • Mediciones de distancia desde la cámara
  • Orientación de la cabeza (ángulos Yaw, Pitch, Roll)

Finalidad del Uso de TrueDepth

Los datos de TrueDepth se utilizan exclusivamente para:

  • Verificar que una persona física real está presente (no una fotografía, video pregrabado o máscara)
  • Prevenir acceso no autorizado al campus universitario mediante suplantación de identidad
  • Cumplir con los protocolos de seguridad institucionales
  • Proteger la seguridad de la comunidad universitaria

Proceso de Detección de Vida

Liveness Activo (en dispositivo):

  1. TrueDepth API procesa datos faciales completamente en el dispositivo del usuario
  2. Se verifica movimiento ocular, parpadeo y orientación de cabeza en tiempo real
  3. Se genera un resultado booleano (persona real: sí/no)
  4. Los datos biométricos crudos se descartan inmediatamente

Liveness Pasivo (en servidor):

  1. Se envía un vector de características faciales (representación matemática) al servidor
  2. Un modelo de aprendizaje automático analiza el vector para detección de vida pasiva
  3. NO se transmiten ni almacenan imágenes faciales
  4. El vector se descarta después de la verificación

Procesamiento y Almacenamiento de Datos

Procesamiento en Dispositivo

TODOS los datos de TrueDepth se procesan íntegramente en su dispositivo en tiempo real. Una vez completada la verificación de vida, los datos biométricos crudos se descartan inmediatamente. La aplicación NUNCA almacena datos biométricos sin procesar.

Información Transmitida a Servidores UDLAP

La aplicación transmite únicamente:

  • Resultado de verificación de vida (booleano: verificado sí/no)
  • Vector de características faciales (representación matemática, NO imágenes)
  • Marca de tiempo del intento de autenticación
  • ID de usuario asociado al intento

Información Almacenada en Servidores

Los servidores de UDLAP almacenan únicamente:

  • Resultado de la autenticación (exitosa/fallida)
  • Marca de tiempo del intento
  • Identificador de usuario
  • Registro de auditoría de acceso al campus

Información que NUNCA se Almacena ni Comparte

  • Mapas de profundidad facial o modelos 3D
  • Datos crudos del sensor TrueDepth
  • Imágenes o fotografías faciales
  • Coordenadas de seguimiento ocular
  • Datos de posición de cabeza sin procesar
  • Plantillas biométricas reconstruibles

Compartir con Terceros

NO se comparten datos de TrueDepth ni información biométrica con terceros. Todo el procesamiento biométrico ocurre localmente en su dispositivo o en servidores seguros de UDLAP. Solo resultados booleanos de verificación (verificado/no verificado) se registran en sistemas institucionales.

Otros Datos Personales Recopilados

Además de la información relacionada con TrueDepth, eUDLAPx recopila:

  • Datos de autenticación: Credenciales institucionales (validadas mediante Microsoft MSAL)
  • Datos de acceso: Registros de entrada y salida del campus, códigos QR escaneados, ubicación de puntos de acceso
  • Datos de dispositivo: Identificador del dispositivo, modelo, versión de iOS, token de notificaciones push (Firebase FCM)
  • Datos de perfil: Nombre, matrícula/ID institucional, rol (estudiante/empleado), fotografía de perfil institucional

Estos datos se utilizan exclusivamente para proporcionar funcionalidades de seguridad y acceso al campus.

Retención de Datos

Los resultados de verificación de vida y registros de acceso se conservan durante el tiempo necesario para cumplir con protocolos de seguridad y auditoría institucionales, en cumplimiento con la Ley Federal de Protección de Datos Personales en Posesión de los Particulares (LFPDPPP).

Los plazos de conservación no excederán lo necesario para el cumplimiento de las finalidades descritas. Una vez cumplidas, y cuando no exista disposición legal que establezca lo contrario, UDLAP procederá a la cancelación de sus datos personales.

Medidas de Seguridad

UDLAP implementa medidas de seguridad técnicas, físicas y administrativas para proteger sus datos personales:

  • Encriptación: Todas las comunicaciones entre la aplicación y servidores utilizan HTTPS/TLS
  • Almacenamiento seguro: Tokens de autenticación almacenados en Keychain de iOS
  • Control de acceso: Acceso limitado a datos personales solo para personal autorizado
  • Auditorías: Revisión periódica de medidas de seguridad y cumplimiento
  • Secure Enclave: Datos biométricos procesados en enclave seguro del dispositivo

Derechos ARCO (Acceso, Rectificación, Cancelación y Oposición)

Como titular de datos personales, usted tiene derecho a:

  • Acceder a sus datos personales en posesión de UDLAP
  • Rectificar datos inexactos o incompletos
  • Cancelar sus datos cuando considere que no se requieren para finalidad alguna
  • Oponerse al tratamiento de sus datos para fines específicos

Para ejercer sus derechos ARCO y conocer los procedimientos aplicables, consulte el Aviso de Privacidad Institucional de UDLAP, donde se detallan los mecanismos, requisitos y plazos para el ejercicio de estos derechos.

Revocación del Consentimiento

Puede revocar su consentimiento para el tratamiento de sus datos personales. Para conocer el procedimiento aplicable, consulte el Aviso de Privacidad Institucional de UDLAP.

Nota Importante: El uso de TrueDepth API es requerido para la funcionalidad de seguridad de eUDLAPx. Si revoca su consentimiento para el procesamiento de datos biométricos, no podrá utilizar la aplicación para acceso al campus y deberá utilizar métodos alternativos de identificación proporcionados por la universidad.
